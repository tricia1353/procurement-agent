# 采购成本与供应商搜索智能体

# 01 产品总览

## 🎯 使用目标

这份操作手册用于指导用户完整使用“采购成本与供应商搜索智能体”。

本产品面向采购、供应链、硬件研发等场景，帮助用户完成两类高频工作：

- **采购成本分析**：输入物料型号，查询产品、价格梯度、库存、交期、封装、RoHS、替代包装等信息，并基于选中的产品生成采购建议。

- **供应商搜索**：输入物料需求、当前痛点、采购要求和期望地区，搜索供应商并整理风险、信用、优势、可供应产品等信息，同时为选中的供应商生成面谈问题清单。

产品整体包含两个主界面：

| 功能入口 | 用途 |
|---|---|
| 采购成本分析 | 查询物料价格与产品信息，进行产品对比，生成采购建议 |
| 供应商搜索 | 根据采购需求搜索供应商，生成面谈问题清单 |

---

## 📑 推荐使用方式

你可以根据自己的目标选择不同路径：

| 使用方式 | 适合人群 | 怎么做 |
|---|---|---|
| 快速体验 | 想先看看页面和基础流程 | 启动前后端后直接使用，未配置 Token 时也能体验部分降级功能 |
| 完整体验 | 想体验采购建议、供应商实时搜索、面谈问题生成 | 启动服务后，在页面右上角配置百度 AI Studio Token |
| 业务演示 | 想向团队展示完整链路 | 按第 03 章依次演示采购成本分析、供应商搜索、面谈问题导出 |

> 提示：如果暂时没有配置大模型 Token，供应商搜索和面谈问题仍有基础兜底能力；但采购建议、实时供应商搜索效果会受到限制。

# 02 使用前准备

## 2.1 准备运行环境

本产品需要同时启动后端服务和前端页面。

| 环境 | 用途 | 建议版本 |
|---|---|---|
| Python | 运行 FastAPI 后端服务 | Python 3.10+ |
| Node.js | 运行 React/Vite 前端页面 | Node.js 16+ |
| 浏览器 | 打开产品页面 | Chrome / Edge / Safari 均可 |

---

## 2.2 项目目录

项目根目录为：

```Plain Text
procurement-agent/
```

主要目录说明：

| 路径 | 用途 |
|---|---|
| `backend/` | 后端服务，包含 FastAPI 接口、Mouser 查询、供应商搜索、面谈问题生成等逻辑 |
| `frontend/` | 前端页面，包含采购成本分析页、供应商搜索页、设置弹窗等 |
| `PRD.md` | 产品需求文档 |
| `操作手册.md` | 当前操作手册 |

---

## 2.3 配置后端环境变量

后端可通过 `.env` 配置外部服务。

在 `backend/` 目录下创建或检查 `.env` 文件：

```env
LLM_ACCESS_TOKEN=your_baidu_aistudio_token
TIANYANCHA_API_KEY=your_tianyancha_key
MOUSER_API_KEY=your_mouser_api_key
HOST=0.0.0.0
PORT=8000
```

各配置项说明：

| 配置项 | 是否必填 | 作用 |
|---|---|---|
| `LLM_ACCESS_TOKEN` | 可选但推荐 | 用于采购建议、供应商实时搜索、面谈问题生成 |
| `TIANYANCHA_API_KEY` | 可选 | 用于查询供应商企业资质和信用评分 |
| `MOUSER_API_KEY` | 推荐 | 用于查询 Mouser 实时产品、价格、库存信息 |
| `HOST` | 可选 | 后端监听地址，默认 `0.0.0.0` |
| `PORT` | 可选 | 后端端口，默认 `8000` |

> 如果不配置 `LLM_ACCESS_TOKEN`，系统会使用推荐供应商和模板问题作为兜底。

---

## 2.4 启动后端服务

打开终端，进入后端目录：

```bash
cd procurement-agent/backend
```

启动 FastAPI 服务：

```bash
python3 -m uvicorn main:app --reload --port 8000
```

看到类似以下信息，说明后端启动成功：

```Plain Text
Uvicorn running on http://127.0.0.1:8000
Application startup complete.
```

可以在浏览器或终端访问健康检查：

```Plain Text
http://localhost:8000/api/health
```

正常返回示例：

```json
{
  "status": "ok",
  "llm_configured": false,
  "tianyancha_configured": false,
  "version": "1.0.0"
}
```

---

## 2.5 启动前端页面

打开新的终端窗口，进入前端目录：

```bash
cd procurement-agent/frontend
```

启动前端开发服务：

```bash
npm run dev -- --host 127.0.0.1
```

看到类似以下信息，说明前端启动成功：

```Plain Text
VITE ready
Local: http://127.0.0.1:3000/
```

在浏览器打开：

```Plain Text
http://localhost:3000
```

页面左上角应显示：

```Plain Text
采购成本与供应商搜索智能体
```

顶部导航应只包含两个入口：

```Plain Text
采购成本分析
供应商搜索
```

# 03 实操步骤

## 3.1 第一步：配置 API Key

应用启动后，建议先配置 API Key。

### 3.1.1 打开设置弹窗

1. 打开 `http://localhost:3000`
2. 点击页面右上角的设置按钮
3. 页面会弹出“API 设置”窗口

设置窗口包含两个配置项：

| 配置项 | 用途 |
|---|---|
| 百度 AI Studio | 用于采购建议和面谈问题生成 |
| 天眼查 API | 用于查询供应商企业资质、信用评分（可选） |

---

### 3.1.2 配置百度 AI Studio Token

如果你需要使用采购建议、供应商实时搜索和面谈问题生成，建议配置百度 AI Studio Token。

操作步骤：

1. 点击“获取 Token”
2. 登录百度 AI Studio
3. 进入个人中心，复制 Access Token
4. 回到应用设置弹窗
5. 粘贴到“百度 AI Studio”输入框
6. 点击“保存设置”

保存后，Key 会存储在本地浏览器中。

> 注意：Token 仅保存在浏览器 localStorage 中，不会显示在页面正文中。

---

### 3.1.3 配置天眼查 API Key（可选）

如果你需要更完整的供应商企业信息，可以配置天眼查 API Key。

操作步骤：

1. 点击“获取 Key”
2. 登录天眼查开放平台
3. 获取 API Key
4. 回到应用设置弹窗
5. 粘贴到“天眼查 API”输入框
6. 点击“保存设置”

如果暂时没有天眼查 Key，也可以继续使用产品；系统会使用已有推荐数据和基础企业信息。

---

## 3.2 第二步：使用采购成本分析

采购成本分析适合用于回答这类问题：

- 某个物料现在有哪些可买产品？
- 不同产品的价格梯度、库存、交期如何？
- 哪个产品更适合当前采购？
- 是否存在库存、交期、封装、RoHS 或替代包装风险？

---

### 3.2.1 进入采购成本分析页面

1. 打开首页
2. 点击顶部导航中的“采购成本分析”
3. 页面显示“采购成本分析”搜索卡片

你会看到一个物料型号输入框和“查询”按钮。

---

### 3.2.2 输入物料型号

在输入框中输入物料型号，例如：

```Plain Text
STM32F103C8T6
```

也可以点击页面中的快捷查询按钮：

```Plain Text
STM32F103C8T6
ESP32-WROOM-32
LM7805
ATmega328P
```

---

### 3.2.3 点击查询

点击“查询”按钮后，系统会：

1. 将型号转为大写
2. 请求后端 `/api/price/search`
3. 查询 Mouser 实时产品数据
4. 返回产品列表、价格梯度、库存和交期信息
5. 在页面展示查询结果

查询中按钮文案会变为：

```Plain Text
查询中...
```

---

### 3.2.4 查看数据来源

查询完成后，页面会显示数据来源，例如：

```Plain Text
数据来源: Mouser API (实时数据)  已配置  · 找到 3 个产品
```

如果 Mouser 未配置，可能显示：

```Plain Text
数据来源: Mouser API 未配置  未配置
```

此时需要检查后端 `.env` 中的 `MOUSER_API_KEY`。

---

### 3.2.5 查看产品列表

有查询结果时，页面会显示“产品列表”。

每个产品卡片包含以下信息：

| 信息 | 说明 |
|---|---|
| 制造商料号 | 例如 `STM32F103C8T6` |
| Mouser 料号 | Mouser 平台内部料号 |
| 制造商 | 例如 ST、Espressif、TI 等 |
| 产品描述 | 产品基础描述 |
| 第一档价格 | 价格梯度中的第一档价格 |
| 库存 | 当前现货库存 |
| 卷带 | 如果支持卷带供应，会显示“卷带”标签 |
| 产品图片 | 如果 Mouser 返回图片，会显示图片 |

---

### 3.2.6 选择产品

点击任意产品卡片即可选中。

选中后：

- 卡片边框变为蓝色
- 背景变为浅蓝色
- 顶部显示“已选 N 个”

再次点击同一个卡片，可以取消选中。

---

### 3.2.7 生成采购建议或对比分析

选择产品后，右上角按钮会启用。

| 已选数量 | 按钮文案 | 用途 |
|---:|---|---|
| 1 个 | 采购建议 | 针对单个产品生成采购判断 |
| 多个 | 对比分析 | 对比多个产品并生成推荐结论 |

点击按钮后，页面会展开“产品分析”或“产品对比”区域。

---

### 3.2.8 查看产品分析 / 产品对比

产品分析区域会以表格方式展示以下信息：

| 对比项 | 说明 |
|---|---|
| 制造商 | 产品制造商 |
| 描述 | 产品描述 |
| 价格梯度 | 展示前 5 档价格 |
| 现货库存 | 当前可采购库存 |
| 在途库存 | 预计到货数量和日期 |
| MOQ / 倍数 | 最小起订量和订购倍数 |
| 交期 | 供应交期 |
| 封装 | 产品封装 |
| 卷带供应 | 是否支持卷带 |
| RoHS | RoHS 合规状态 |
| 替代型号 | 可替代包装或替代料号 |
| 产品链接 | 跳转 Mouser 产品页 |

---

### 3.2.9 查看采购建议

在产品分析表格下方，会显示“采购建议”。

生成过程中会看到以下步骤：

```Plain Text
正在整理产品对比数据...
正在分析价格竞争力...
正在评估供货能力与风险...
正在生成采购推荐方案...
```

生成完成后，系统会以 Markdown 形式展示采购建议。

单个产品通常包含：

```Plain Text
价格评估
供货能力
适用场景
风险提示
采购建议
```

多个产品通常包含：

```Plain Text
价格竞争力
供货能力
适用场景
风险提示
推荐结论
```

---

### 3.2.10 采购成本分析常见问题

#### 情况一：没有产品结果

可能原因：

1. 物料型号输入错误
2. Mouser API 未配置
3. 该型号在 Mouser 中无匹配结果

建议操作：

1. 检查型号拼写
2. 尝试更短或更通用的关键词
3. 检查后端 `.env` 中是否配置了 `MOUSER_API_KEY`

#### 情况二：采购建议生成失败

可能原因：

1. 未配置百度 AI Studio Token
2. Token 已过期
3. 后端 `/api/chat` 调用失败

建议操作：

1. 点击右上角设置按钮
2. 重新填写百度 AI Studio Token
3. 保存后重新点击“采购建议”或“对比分析”

## 3.3 第三步：使用供应商搜索

供应商搜索适合用于回答这类问题：

- 某个物料可以找哪些供应商？
- 当前供应商不稳定，有哪些替代供应商？
- 哪些供应商在指定地区？
- 面谈供应商时应该问哪些问题？

---

### 3.3.1 进入供应商搜索页面

1. 打开首页
2. 点击顶部导航中的“供应商搜索”
3. 页面左侧显示“供应商条件”表单
4. 页面右侧显示结果区

---

### 3.3.2 填写物料信息

在“物料信息”区域填写：

| 字段 | 示例 | 说明 |
|---|---|---|
| 物料型号 | `RF模块 KBD600 433M` | 可填写具体型号 |
| 物料类别 | `RF模块` | 从下拉框选择 |
| 规格参数 | `433MHz, 10mW` | 可填写关键规格 |
| 年用量 | `1000000` | 可填写预计采购规模 |

物料类别可选：

```Plain Text
芯片/IC、液晶/显示屏、面板、PCB、被动元件、连接器、RF模块、非标件、其他
```

---

### 3.3.3 选择当前痛点

在“当前痛点”区域选择当前采购中遇到的问题。

预置选项：

```Plain Text
价格贵、起订量多、交期长、质量不稳定、保供风险、资质不全、服务差
```

操作方式：

1. 点击痛点标签即可选中
2. 再次点击可取消选中
3. 如有其他痛点，可在输入框中填写
4. 点击“添加”加入自定义痛点
5. 已添加的痛点可以点击右侧 X 删除

---

### 3.3.4 选择采购要求

在“采购要求”区域选择对供应商的要求。

预置选项：

```Plain Text
可提供参考规格书、接受替代品牌、小批量订单、账期支持、技术支持、样品测试
```

操作方式与“当前痛点”一致。

---

### 3.3.5 填写当前供应商（可选）

如果你希望寻找当前供应商的替代方案，可以填写“当前供应商”。

示例：

```Plain Text
粤盛
```

填写后页面会提示：

```Plain Text
将搜索可替代「粤盛」的其他供应商
```

如果没有当前供应商，可以留空。

---

### 3.3.6 选择期望地区

可选择供应商所在地区。

预置选项：

```Plain Text
深圳、上海、东莞、杭州、苏州、南京、北京、成都、不限
```

说明：

- 可以选择多个地区
- 选择“不限”表示不限制地区
- 提交给后端时，“不限”不会作为地区条件传递

---

### 3.3.7 点击开始搜索

填写完成后，点击底部“开始搜索”。

搜索过程中，右侧会显示四个阶段：

```Plain Text
搜索供应商
整理结果
查询企业信息
生成列表
```

页面底部会提示：

```Plain Text
预计需要 30-60 秒，请耐心等待...
```

---

### 3.3.8 查看供应商搜索结果

搜索完成后，页面会显示：

```Plain Text
找到 N 家供应商
```

如果输入了物料型号，会显示：

```Plain Text
找到 N 家供应商（针对 xxx）
```

页面还会显示搜索来源，例如：

```Plain Text
搜索来源: 联网搜索  天眼查
```

如果未配置大模型 Token，系统会使用推荐供应商作为参考，并显示提示信息。

---

### 3.3.9 切换表格 / 卡片视图

结果区右上角有两个按钮：

```Plain Text
表格
卡片
```

| 视图 | 适合场景 |
|---|---|
| 表格 | 快速横向比较多个供应商 |
| 卡片 | 查看单个供应商更完整的信息 |

默认使用表格视图。

---

### 3.3.10 查看表格结果

表格包含以下列：

| 列名 | 说明 |
|---|---|
| 选择框 | 勾选供应商，用于后续生成面谈问题 |
| 供应商名称 | 显示名称、风险等级、信用分 |
| 地点 | 供应商所在地区，可点击排序 |
| 主要优势 | 展示供应商优势，最多显示前 2 项 |
| 可供应产品 | 展示供应产品，最多显示前 2 项 |
| 已服务客户 | 展示客户案例，最多显示前 2 项 |
| 信息来源 | 显示数据来源 |

说明：当前表格不展示“匹配度”列。

---

### 3.3.11 选择供应商

勾选表格左侧的复选框，即可选择供应商。

选择后，表格底部会出现蓝色操作条：

```Plain Text
已选择 N 家供应商
清空选择
生成面谈问题 ↓
```

按钮说明：

| 按钮 | 作用 |
|---|---|
| 清空选择 | 取消所有已选供应商 |
| 生成面谈问题 ↓ | 滚动到下方的面谈问题生成器 |

---

## 3.4 第四步：生成供应商面谈问题

面谈问题功能用于帮助采购人员准备供应商沟通问题。

---

### 3.4.1 进入面谈问题区域

有两种方式进入：

1. 在供应商表格中勾选供应商后，点击“生成面谈问题 ↓”
2. 直接向下滚动到页面下方的面谈问题区域

如果没有选择供应商，页面会提示：

```Plain Text
请先勾选供应商
勾选后可生成面谈问题清单
```

---

### 3.4.2 选择重点关注领域

选择供应商后，会显示“重点关注领域”。

默认选中：

```Plain Text
价格、交期、质量
```

可选项：

```Plain Text
价格、起订量、交期、质量、资质、服务、技术支持
```

你可以根据本次面谈目标进行调整。

例如：

| 场景 | 建议关注领域 |
|---|---|
| 供应商报价偏高 | 价格、起订量、账期相关问题 |
| 交付不稳定 | 交期、保供风险、替代方案 |
| 新供应商导入 | 资质、质量、服务、技术支持 |
| 研发选型阶段 | 技术支持、样品测试、规格匹配 |

---

### 3.4.3 点击生成面谈问题

点击按钮：

```Plain Text
生成 N 家供应商的面谈问题
```

生成中按钮会显示：

```Plain Text
生成中...
```

系统会根据以下信息生成问题：

1. 已选供应商名称
2. 物料型号
3. 物料类别
4. 年用量
5. 当前痛点
6. 采购要求
7. 重点关注领域

---

### 3.4.4 查看面谈问题清单

生成完成后，页面显示“面谈问题清单”。

问题会按供应商分组展示，例如：

```Plain Text
深圳华强电子（5 个问题）
```

点击供应商名称可以展开或收起。

每个问题包含：

| 内容 | 说明 |
|---|---|
| 问题类别 | 如企业基本情况、产品与质量、商务条款、风险点等 |
| 优先级 | 高 / 中 / 低 |
| 问题正文 | 具体面谈问题 |
| 背景 | 为什么要问这个问题 |

---

### 3.4.5 复制单条问题

每条问题右侧有复制按钮。

点击后，该问题会复制到剪贴板，可直接粘贴到：

- 微信
- 邮件
- 飞书
- 面谈纪要
- 供应商沟通记录

---

### 3.4.6 导出面谈问题清单

点击右上角“导出”，系统会下载 `.txt` 文件。

文件名格式：

```Plain Text
面谈问题_物料型号_日期.txt
```

例如：

```Plain Text
面谈问题_STM32F103C8T6_2026-06-12.txt
```

导出的文件包含：

1. 标题
2. 生成时间
3. 物料型号与类别
4. 各供应商问题
5. 通用问题

---

### 3.4.7 未配置 Token 时会怎样？

如果没有配置百度 AI Studio Token，系统不会直接报错。

此时会使用内置模板生成基础面谈问题，通常包括：

```Plain Text
企业基本情况
产品与质量
商务条款
风险点
服务支持
当前痛点追问
```

如果配置了 Token，系统会根据供应商和物料信息生成更有针对性的问题。

# 05 常见问题

## Q1：页面打不开怎么办？

请依次检查：

1. 前端是否启动：`http://localhost:3000`
2. 后端是否启动：`http://localhost:8000/api/health`
3. 终端是否有报错
4. 端口是否被占用

如果前端页面返回 Vite 报错，请优先看报错提示中的文件名和行号。

---

## Q2：页面显示空白怎么办？

常见原因：

1. Vite 开发服务器没有启动
2. 浏览器打开了错误端口
3. 前端编译失败
4. Tailwind 自定义类未配置

建议操作：

```bash
cd procurement-agent/frontend
npm run build
```

如果构建失败，根据终端错误修复。

---

## Q3：提示 `shadow-glow class does not exist` 怎么办？

说明 Tailwind 没有识别自定义阴影类。

检查 `frontend/tailwind.config.js` 中是否存在：

```js
boxShadow: {
  soft: '0 18px 45px rgba(15, 23, 42, 0.08)',
  glow: '0 12px 34px rgba(37, 99, 235, 0.18), 0 0 24px rgba(6, 182, 212, 0.16)',
  'glow-lg': '0 18px 44px rgba(37, 99, 235, 0.24), 0 0 34px rgba(6, 182, 212, 0.22)'
}
```

如果配置已存在，但页面仍报错，重启前端服务：

```bash
npm run dev -- --host 127.0.0.1
```

---

## Q4：价格查询没有结果怎么办？

可能原因：

1. 物料型号写错
2. Mouser 没有该型号
3. 后端没有配置 Mouser API Key
4. 网络请求失败

建议尝试：

```Plain Text
STM32F103C8T6
ESP32-WROOM-32
LM7805
```

如果仍无结果，检查后端日志。

---

## Q5：采购建议生成失败怎么办？

可能原因：

1. 没有配置百度 AI Studio Token
2. Token 已过期
3. 后端 `/api/chat` 出错
4. 网络超时

解决步骤：

1. 点击右上角设置按钮
2. 重新粘贴百度 AI Studio Token
3. 点击保存设置
4. 刷新页面
5. 重新生成采购建议

---

## Q6：供应商搜索结果看起来是推荐数据，不是实时搜索怎么办？

如果没有配置百度 AI Studio Token，系统会使用推荐供应商兜底。

解决步骤：

1. 点击右上角设置按钮
2. 配置百度 AI Studio Token
3. 保存设置
4. 重新点击“开始搜索”

---

## Q7：生成面谈问题没有反应怎么办？

请检查：

1. 是否已经勾选供应商
2. 是否至少选择了一个重点关注领域
3. 后端是否运行正常
4. 浏览器控制台是否有接口报错

如果未配置 Token，系统仍应返回模板问题；如果完全没有反应，优先检查 `/api/supplier/interview-questions` 接口是否可用。

---

## Q8：如何向开发助手描述问题？

遇到问题时，不要只说“坏了”或“没反应”。建议按下面格式描述：

```Plain Text
我刚刚做了什么：
我预期看到什么：
实际发生了什么：
页面或终端报错：
我已经尝试过什么：
```

示例：

```Plain Text
我刚刚在供应商搜索里填写了 RF模块，点击开始搜索。
我预期看到供应商列表。
实际页面停在加载中。
后端终端报错 500。
我已经刷新页面并重新点击过一次。
```

这样更容易快速定位问题。
